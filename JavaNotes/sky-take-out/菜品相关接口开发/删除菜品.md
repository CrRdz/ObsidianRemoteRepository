# 需求分析和设计

业务规则
- 可以一次删除一个菜品 也可以批量删除菜品
- 起售中的菜品不能删除
- 被套餐关联的菜品不能删除
- 删除菜品后关联的口味数据也需要被删除掉
## 接口设计
- 基本信息
	- Path:/admin/dish
	- Method:DELETE
- 请求参数
	Query

| 参数名称 | 是否必须 | 示例    | 备注           |
| ---- | ---- | ----- | ------------ |
| ids  | 是    | 1，2，3 | 菜品id 之间用逗号分隔 |
- 返回数据

| 名称   | 类型     | 是否必须 | 默认值 | 备注  | 其他信息 |
| ---- | ------ | ---- | --- | --- | ---- |
| code | number | 必须   |     |     |      |
| data | string | 非必须  |     |     |      |
| msg  | string | 非必须  |     |     |      |
