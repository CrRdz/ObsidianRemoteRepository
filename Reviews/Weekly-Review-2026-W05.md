# 📅 Week 5 Learning Review

> Jan 26 - Feb 01, 2026

---

### 1. 📊 本周概览
本周主要围绕 Java 后端项目 `sky-take-out` 的实战开发进行学习，核心主题是 **Redis** 的应用与 **菜品管理模块** 的 CRUD 功能实现。学习强度适中，通过具体接口开发深入理解了 Redis 在缓存和状态管理中的实践，以及项目中的一些通用技术点。

### 2. 📚 学习内容

## Redis
- 学习了 Redis 在项目中的实际应用，主要用于 **缓存店铺营业状态**，以提升查询性能。
- 掌握了使用 **Lua 脚本** 来保证 `查询-更新` 操作的原子性，避免并发问题。
- 了解了 Lombok 工具在简化 Java Bean 代码（如自动生成 getter/setter）方面的便捷性。

## 菜品管理模块
- 系统开发了菜品的 **分页查询、新增、修改、删除** 等全套 CRUD 接口。
- 实现了 **公共字段自动填充** 功能（如 `create_time`, `update_time`），利用注解和元数据对象处理器在插入或更新时自动写入操作人和时间。
- 完成了与菜品紧密关联的 **分类模块功能**，为菜品提供了必要的分类数据支持。

## 项目框架与搭建
- 回顾并实践了项目的前置搭建步骤，包括基础依赖配置和项目结构初始化。
- 巩固了 **员工信息编辑** 等基础管理功能的实现逻辑，作为项目开发的复习。

### 3. 💡 本周金句
- “使用 Lua 脚本可以确保多个 Redis 命令的原子执行，是解决并发场景下状态一致性的有效方案。”
- “公共字段自动填充通过 AOP 思想，将通用逻辑抽离，避免了在每一个业务代码中进行重复设置。”
