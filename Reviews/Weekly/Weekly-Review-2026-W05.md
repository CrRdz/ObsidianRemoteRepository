### 1. 📊 Weekly Overview
本周专注于**Java Web**项目（`sky-take-out`）的后端功能开发，并引入了 **Redis** 来优化特定业务场景。学习强度中等，但实践性很强，重点在于将新学的 Redis 知识立即应用到具体的业务接口开发中，加深了对缓存作用和项目架构的理解。

### 2. 📚 Content

## Java Web (sky-take-out 项目)
- **项目搭建与通用功能**：完成了项目的前置环境搭建，并实现了**公共字段自动填充**，利用`MetaObjectHandler`在`insert/update`时自动写入创建人、时间等字段。
- **核心业务模块开发**：深入开发了**分类模块**和**菜品管理模块**。具体实现了菜品的**分页查询**、**新增**、**修改**、**删除**以及员工信息编辑等全套**CRUD**接口，对Spring Boot项目结构更熟悉了。

## Redis
- **缓存应用实践**：将 Redis 应用于**店铺营业状态**管理，替代了直接查询数据库的方式，显著提升了状态读取的性能和响应速度。
- **技术栈深化**：学习了如何在Java中集成并使用Redis，特别是**LuaScript**来保证**原子性**操作。同时，在项目中更熟练地使用 **Lombok** 来简化实体类的`getter/setter`和`builder`代码。

### 3. 💡 Weekly Insight Log
将业务状态管理从数据库迁移到 Redis，不仅仅是换了个存储位置，更是一种**架构思维的转变**——从“持久化存储”转向了“高性能缓存”，这是应对高并发场景的经典策略。
