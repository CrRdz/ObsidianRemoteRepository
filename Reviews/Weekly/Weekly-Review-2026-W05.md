# 📚 Learning Content

## Redis 应用与优化
- 学习了 **Redis** 在项目中的两种核心用法：
  - **数据缓存**：用于存储高频访问但更新不频繁的数据，如店铺营业状态，以减轻数据库压力。
  - **原子操作与 Lua Script**：通过 Lua 脚本确保复杂操作（如状态判断与更新）的原子性，避免并发问题。
- 接触了工具 **Lombok**，用于通过注解简化 Java Bean 的代码（如自动生成 `getter/setter`），提升开发效率。

## 项目业务功能模块开发
- 深入开发了 **菜品管理** 核心 CRUD 功能：
  - **新增菜品**：涉及多表数据插入（菜品基本信息、口味）。
  - **菜品分页查询**：通常需要关联查询并处理结果集封装。
  - **修改/删除菜品**：注意业务逻辑完整性，如删除前的状态校验。
- 实现了 **公共字段自动填充**：利用 `MetaObjectHandler` 或自定义切面，在插入/更新时自动填充 `create_time`, `update_time`, `create_user`, `update_user` 等字段，统一且规范。
- 搭建了项目 **前置基础结构**，并完成了 **员工编辑** 和 **分类管理** 等基础模块，为后续复杂功能开发打下基础。

# 💡 Key Insights
> “Redis 不仅仅是简单的缓存，结合 Lua Script 可以将其升级为支持原子性复杂逻辑的‘轻量级业务逻辑执行引擎’，这是应对高并发场景下数据一致性的一个利器。”
